## Mission

Produis un dossier produit exploitable par les equipes design, frontend, backend, data et go-to-market.

## Sortie attendue

1. Carte des personas:
   - visiteur public
   - prospect entreprise
   - administrateur tenant
   - agent call center
   - superviseur
   - developpeur API
   - equipe support interne
2. Parcours par route:
   - objectif utilisateur
   - message marketing
   - CTA principal
   - donnees capturees
   - microservices impliques
   - evenements emis
3. Backlog epique:
   - epique
   - user stories
   - criteres d'acceptation
   - dependances
   - niveau de risque
4. Matrice MVP / V1 / V2.
5. Liste des questions bloquantes a poser au client.

## Contraintes

- Ne suppose pas que toutes les fonctionnalites doivent etre dans le MVP.
- Separe clairement site marketing, application SaaS, portail developpeur et consoles CPaaS.
- Chaque fonctionnalite doit pouvoir etre rattachee a une solution autonome.
